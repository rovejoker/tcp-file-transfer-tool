# TCP 文件传输工具

一个基于 TCP 协议的文件传输工具，支持大文件分块传输、断点续传、心跳检测和安全防护功能。

## 功能特性

### 核心功能
- ✅ **文件上传**：客户端可上传本地文件到服务器
- ✅ **文件下载**：客户端可从服务器下载指定文件
- ✅ **文件列表**：客户端可查看服务器上的文件列表
- ✅ **大文件分块传输**：支持超大文件的分块传输，避免内存溢出
- ✅ **断点续传**：支持上传和下载的断点续传，网络中断后可恢复传输

### 网络特性
- ✅ **心跳检测**：定期检测连接状态，及时发现网络异常
- ✅ **自动重连**：连接断开后自动尝试重新连接（指数退避策略）
- ✅ **长连接设计**：支持同一客户端多次上传/下载操作

### 安全防护
- ✅ **单IP连接数限制**：限制单个IP的最大并发连接数（默认5个）
- ✅ **路径穿越防护**：禁止 `../` 等路径穿越字符，保护系统文件安全
- ✅ **文件名长度限制**：限制文件名最大长度为120字符
- ✅ **文件哈希校验**：SHA256 校验确保文件传输完整性
- ✅ **文件大小限制**：可配置最大上传文件大小

### 用户体验
- ✅ **实时进度条**：显示上传/下载进度和传输速度
- ✅ **配置管理**：支持服务器地址、端口、下载目录等配置
- ✅ **中文路径支持**：完美支持中文文件名和路径

## 技术实现

### 协议设计
- **固定包头协议**：采用固定长度包头（16字节），彻底解决TCP粘包问题
- **协议结构**：版本号 + 帧类型 + 数据长度 + 帧序号 + 保留字段
- **帧类型**：JSON帧、二进制帧、分块帧

### 分块传输
- **分块大小**：默认 64KB
- **确认机制**：上传完成后服务端确认，下载时无需逐块确认
- **流式写入**：边接收边写入磁盘，减少内存占用

## 安装与运行

### 环境要求
- Python 3.8+
- Windows / Linux / macOS

### 快速开始

1. **启动服务器**：
```bash
python run.py
```
然后选择 `1` 启动服务器

或者直接运行：
```bash
python scripts/server.py
```

2. **启动客户端**：
```bash
python run.py
```
然后选择 `2` 启动客户端

或者直接运行：
```bash
python scripts/client.py
```

### 使用说明

#### 服务器端
1. 运行服务器脚本
2. 可选修改配置（端口、资源目录、最大文件大小）
3. 服务器启动后监听指定端口（默认 2983）

#### 客户端
1. 运行客户端脚本
2. 可选修改配置（服务器地址、端口、下载目录）
3. 连接到服务器后，选择操作：
   - 1. 上传文件：输入本地文件路径和显示名称
   - 2. 下载文件：输入服务器上的文件名
   - 3. 查看服务器文件列表
   - 4. 返回配置界面
   - 5. 退出

## 配置说明

### 服务器配置 (config/server_config.json)
```json
{
    "server": {
        "port": 2983,
        "resource_dir": "resource",
        "max_file_size_mb": 1024
    }
}
```

### 客户端配置 (config/client_config.json)
```json
{
    "client": {
        "server_host": "127.0.0.1",
        "server_port": 2983,
        "download_dir": "downloads"
    }
}
```

## 安全特性

### 连接数限制
- 限制单个IP最大并发连接数为5个
- 超出限制的连接会被拒绝并记录日志
- 可通过配置调整限制数量

### 路径穿越防护
- 使用 `os.path.basename()` 强制提取纯文件名
- 过滤 `../` 和路径分隔符
- 二次校验最终路径必须在指定目录内

### 文件完整性校验
- 上传/下载时计算 SHA256 哈希值
- 传输完成后比对哈希确保完整性
- 断点续传时自动检测文件完整性

## 心跳机制

- **心跳间隔**：10秒
- **超时次数**：3次
- **重连策略**：指数退避（1s, 2s, 4s, 8s, 16s, 30s）
- **传输暂停**：文件传输期间暂停心跳检测，避免干扰

## 断点续传

### 上传续传
1. 客户端发送上传请求时携带 `resume_from` 参数
2. 服务端检查已存在文件大小并返回断点位置
3. 客户端从断点位置继续上传

### 下载续传
1. 客户端检查本地已下载文件大小
2. 请求时携带断点位置
3. 服务端从断点位置开始发送文件
